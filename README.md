1. КОНЦЕПЦИЯ ИГРЫ
═══════════════════════════════════════════════════════════════════════════════

ЖАНР:
2D Action Roguelike с элементами RPG

ЦЕЛЕВАЯ АУДИТОРИЯ:
• Любители roguelike игр
• Фанаты классических 2D экшенов
• Игроки, ценящие челлендж

ОСНОВНАЯ ИДЕЯ:
Игрок исследует процедурно генерируемые подземелья, сражается с врагами,
собирает лут и пытается выжить как можно дольше. Каждая смерть означает
начало с нуля, но с накопленным опытом игрока.

УНИКАЛЬНЫЕ ОСОБЕННОСТИ:
• Простое управление (WASD + Space)
• Быстрый геймплей
• Система боя с броней
• Умный AI врагов
• Визуальная обратная связь (HP бары)

ВДОХНОВЕНИЕ:
• The Binding of Isaac (геймплей)
• Enter the Gungeon (боевая система)
• Nuclear Throne (динамика)

Тип:          Свободное перемещение

ВРАГИ:
Скорость:     150 пикселей/секунду (37.5% от игрока)
Тип:          AI-управляемое преследование
Коллизии:     Останавливаются при столкновении

ФИЗИКА:

2.3 СИСТЕМА БОЯ
─────────────────────────────────────────────────────────────────────────────

ХАРАКТЕРИСТИКИ:

ИГРОК:
Здоровье (HP):        100
Броня:                10
Урон атаки:           20
Радиус атаки:         150 пикселей
Перезарядка атаки:    0.5 секунды
Тип атаки:            Ближний бой (ручная)

ВРАГ (базовый):
Здоровье (HP):        50
Броня:                5
Урон атаки:           12
Радиус атаки:         120 пикселей
Перезарядка атаки:    1.0 секунда
Тип атаки:            Ближний бой (автоматическая)

ФОРМУЛА УРОНА:
finalDamage = max(1, attackDamage - targetArmor)

Примеры:
• Игрок атакует врага: max(1, 20 - 5) = 15 урона
• Враг атакует игрока: max(1, 12 - 10) = 2 урона
• Минимальный урон всегда 1 (броня не дает полной защиты)



МЕХАНИКА АТАКИ:

ИГРОК:
1. Нажатие Space
2. Проверка cooldown (готова ли атака)
3. Поиск цели в радиусе атаки
4. Нанесение урона ближайшей цели
5. Запуск cooldown таймера

ВРАГ:
1. Автоматическая проверка каждый кадр
2. Если цель в радиусе атаки И cooldown готов
3. Нанесение урона цели
4. Запуск cooldown таймера

ВИЗУАЛЬНАЯ ОБРАТНАЯ СВЯЗЬ:
• Полоса HP над персонажами
• Цвет полосы: зеленый (>66%), желтый (33-66%), красный (<33%)
• Логи в консоль о нанесенном уроне
• Логи о смерти персонажей

2.4 СИСТЕМА AI ВРАГОВ
─────────────────────────────────────────────────────────────────────────────

ПОВЕДЕНИЕ ВРАГА:

СОСТОЯНИЕ: IDLE (Ожидание)
Условие:    Игрок вне радиуса обнаружения
Действие:   Стоит на месте
Переход:    → CHASE когда игрок входит в радиус

СОСТОЯНИЕ: CHASE (Преследование)
Условие:    Игрок в радиусе обнаружения (400 пикселей)
Действие:   Движется к игроку со скоростью 150 пикс/сек
Переход:    → IDLE когда игрок выходит из радиуса
→ ATTACK когда игрок в радиусе атаки

СОСТОЯНИЕ: ATTACK (Атака)
Условие:    Игрок в радиусе атаки (120 пикселей)
Действие:   Атакует каждую секунду (cooldown)
Переход:    → CHASE когда игрок выходит из радиуса атаки

АЛГОРИТМ ПРЕСЛЕДОВАНИЯ:

1. Получить позицию цели из TargetComponent
2. Вычислить вектор направления: direction = target - enemy
3. Вычислить расстояние: distance = length(direction)
4. Если distance <= detectionRadius:
a. Нормализовать направление: normalized = direction / distance
b. Вычислить движение: movement = normalized \* speed \* deltaTime
c. Применить движение: transform->MoveBy(movement)

ОСОБЕННОСТИ AI:
• Прямолинейное преследование (без pathfinding)
• Не обходит препятствия (упирается в стены)
• Всегда знает позицию игрока (нет тумана войны)
• Мгновенная реакция на появление игрока

3.1 КОМПОНЕНТНАЯ СИСТЕМА
─────────────────────────────────────────────────────────────────────────────

ПРИНЦИП:
Entity-Component System (ECS) - каждый игровой объект (GameObject)
состоит из набора компонентов, каждый из которых отвечает за
определенную функциональность.

БАЗОВЫЕ КОМПОНЕНТЫ ДВИЖКА:

Component (базовый класс)
└─ Методы: Update(), Render()
└─ Ссылка на GameObject

TransformComponent
└─ Позиция, поворот, масштаб
└─ Иерархия трансформаций (parent-child)

SpriteRendererComponent
└─ Отображение спрайта
└─ Текстура, размер, flip

RigidbodyComponent
└─ Физическое тело
└─ Кинематический/динамический режим

ColliderComponent
└─ Коллизии
└─ AABB прямоугольник

InputComponent
└─ Обработка ввода
└─ Оси движения (horizontal, vertical)

MovementComponent
└─ Движение на основе ввода
└─ Скорость перемещения

CameraComponent
└─ Камера следящая за объектом
└─ Базовое разрешение

НОВЫЕ КОМПОНЕНТЫ (добавлены в проекте):

TargetComponent
└─ Отслеживание цели
└─ Поиск по имени или прямой ссылке
└─ Кэширование позиции

EnemyMovementComponent
└─ AI движение к цели
└─ Радиус обнаружения
└─ Скорость преследования

CombatStatsComponent
└─ Боевые характеристики
└─ HP, максимальное HP, броня
└─ Методы урона и лечения

AttackComponent
└─ Система атак
└─ Урон, радиус, cooldown
└─ Ручная/автоматическая атака

CombatStatsDisplayComponent
└─ Визуализация HP
└─ Полоса здоровья
└─ Цветовая индикация
